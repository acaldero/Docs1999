Usuarios de una Maquina

       
    LISTADO DE LOS USUARIOS CONECTADOS

          # !!! --- w --- sh users --- QUERY NAMES

            * Lista los usuarios conectados en ese momento a la
              maquina local ( la maquina desde la que se ejecuta el
              comando indicado ) y que comando esta ejecutando
              en ese momento.

          # !!! --- who --- ??? --- ???

            * Lista los usuarios conectados en ese momento y desde
              donde estan conectados.

          # finger <@maquina> (ux,vms)

            * lista los usuarios conectados a esa maquina en ese
              preciso momento

          # !!! --- rusers --- ruser --- ???

            * Lista las maquinas accesibles desde la que estas conectado,
              y de cada maquina, los usuarios conectados en ese momento.
            * Si se queda como "colgado" pulsar control-c para regresar
              al prompt del sistema.

       
    DATOS DE UN USUARIO

          # finger <usuario@maquina> (ux,vms)

            * Visualiza la informacion de un usuario.
            * finger <usuario>  : busca el usuario en la maquina local

          # !!! --- whois <usuario> --- ??? --- ???

            * Intenta mostrar que cuentas tienen como nombre de usuario
              el que se le pasa por parametro.

       
    CAMBIAR ALGUNOS DATOS TUYOS

          ( NOTA:
             La mayoria de los datos presentados por los comandos
             anteriores, estan en alguno de los "ficheros especiales",
             ver esa seccion para mas detalle. )

          # chfn (ux)

            * Cambia el nombre del propietario de la cuenta,
              informacion esta que sale con la orden finger.

               mensajes los demas usuarios usando write y talk.

Comunicacion entre Maquinas

       
    ACCESO REMOTO A TU CUENTA

         # telnet <maquina>  (dos,ux,vms)

           * Permite conectarte a otro ordenador, como un usuario
             en una terminal del mismo.
           * En el "argot" informatico, esa maquina se llama HOST.

         # tn3270 <host> (dos,ux,vms)

           * Permite conectarte a un ordenador con VM-ESA(host), como
             terminal 3270.
           * Muchas versiones de telnet tienen un parametro para actuar
             como este programa.

          Ej. en el telnet de AIX, "telnet -e 3270"

         # rlogin -l <Usuario_Anfitrion> <maquina> (Unx) :::
              --- rlogin/username=<Usuario_Anfitrion> <maquina> (VMS)

           * Permite conectarte a la cuenta de un usuario de una maquina
             ( puede ser la misma ) como si de el mismo se tratara sin
               tener que introducir password ( antes , ese usuario tiene
       que darte de alta en su fichero .rhosts ).
       ( para mas informacion del fichero .rhosts ver seccion
                 ficheros especiales )
       El parametro -l o /Username es opcional. Si no se indica, se
       asume que queremos entrar en nuestra cuenta, de otra maquina.

         # psi  (vms)

           * Permite conectarte a otro ordenador, como un usuario
           * Para mas informacion :
               -  PSI

       
    ACCESO A LOS RECURSOS DE OTROS ORDENADORES

         # ftp <maquina>  (dos,ux,vms)

           * Permite intercambio de ficheros entre dos cuentas de
             un mismo ordenador o de ordenadores diferentes.
           * Ante la duda, traerse los fichero en binario. Exite
             un truco para "convertirlos" a ascii.
             (ver fichero trucos)
           * Siglas de "File Transfer Protocol".
           * Si el ftp no es interactivo, en algunos sistemas el
             caracter @ lo interpreta como enter. Que hacer?
             (ver seccion trucos)
           * Para mas informacion :
               -  Ftp

         # fsp <maquina>  (dos,ux,vms,system7)

           * Intercambio de ficheros entre ordenadores, pero te conectas
             sin necesitar password. Te conectas a un "servidor fsp".
           * No necesita linea dedicada; para cada orden te conectas,
             se ejecuta y se desconecta.
           * Las fuentes son publicas con lo que las puedes compilar
             para cualquier maquina ( incluso las fuentes del servidor )
           * Siglas de "File Service Protocol".(?)
           * Ojo, con fsp te conectas a un servidor fsp, no ftp
             idem con ftp ( no sirve uno como otro, aunque tenga
             funcion parecida )
           * Ver en seccion programa, apartado fsp.

Navegadores WWW

 
        Son programas que presenta 'hojas' de texto, en las cuales hay
     unas palabras subrayadas. Si se quiere saber mas de esa palabra,
     se seleciona con el raton esa opcion. En poco tiempo se muestra
     otra pagina con informacion referente a esa palabra. De esta
     forma se puede viajar por un "periodico" casi infinito.
     Las hojas se llaman "paginas web" ;
     Las paginas web forman una telaran~a mundial, conocida con el
     nombre de "Word-Wide-Wed" o bien "WWW".
        Por dentro, la informacion de cada hoja se trae con un protocolo
     de comunicaciones (http,ftp,news,gopher...), creandose una conexion
     que se cierra al recibir la pagina. 

       # NetScape (win,ux)

         * Presenta las paginas web, en modo grafico.
           El mas conocido.

       # Xmosaic (ux)

         * Presenta las paginas web, en modo grafico.
           El primero conocido.

       # lynx (ux,vms)

         * Presenta las paginas web, en modo TEXTO.
           Muy usado por existir muchos terminales vt100.

       # explorer (win95)

         * En modo grafico...

VMSMail

 

     Es el sistema de mensajeria que viene incluido con VMS,
  solo sirve para enviar mensajes localmente o utilizando DECnet
  (software de comunicaciones de DEC), este sistema exige que la
  maquina destino este funcionando y la entrega del mensaje se hace
  en el momento.

     Sirve para enviar mensajes a los usuarios de la maquina y a
  otras de la facultad que tienen DECnet (ZAPE, GOOFY, OBELIX, ...) el
  formato de las direcciones en este caso es "HOST::USUARIO" donde HOST
  es el nombre de la maquina y USUARIO el "username" de cada uno.

     Utilizando la pasarela de correo que tenemos instalada en la
  facultad, este sistema nos permite, aunque de una forma un poco
  mas incomoda, enviar mensajes a cualquier direccion internet, el
  formato en este caso sera:

     IN::"usuario@direccion.internet"
     (en este caso las dobles comillas son obligatorias y
      parte de la direccion).

     Este sistema de correo tambien esta organizado en carpetas, No
  guarda copia de los mensajes enviados (pero los mensajes se pueden
  enviar con copia a uno mismo de forma automatica). Los mensajes se
  reciben siempre en la carpeta NEWMAIL y cuando se leen pasan de
  forma automatica a la carpeta MAIL que es donde se trabaja normalmente.

     Al entrar si hay mensajes pendientes nos situa en la carpeta
  NEWMAIL y si no en la MAIL.

     Este sistema se activa con el comando MAIL, si se ha definido un
  directorio de trabajo se almacenaran en el todos los ficheros del
  MAIL (MAIL.MAI y los correspondientes a los mensajes), si no lo
  hara en nuestro directorio por defecto.

     Los comandos, que se pueden abreviar siempre que no sean ambiguos,
  mas habituales de este sistema son:

News (zipi)

 
 
  Las noticias (news) se parecen a los mails;
Las de un tema estan agrupadas en un una carpeta (folder), y
Las carpetas relacionadas, se agrupan a su vez en grupos.
Una carpeta se identifica por :

    <nombre grupo>.<nombre carpeta>

  Es muy util, porque es usado para notificar a los usuarios los
cambios en el sistema, avisos de los departamentos o avios en
general de interes publico.

  Para empezar, teclear "news" en el prompt del sistema para
entrar, con lo cual deberia aparecer el prompt del news (en zipi
hay que esperar y esperar). Los comandos siguientes son los mas
habituales :

      help            ---> para pedir ayuda. Estructura normal de
   Vms.

      dir             ---> lista las news de la carpeta actual
    apareceran varias columnas, como el
    numero de orden, titulo de la news
    quien la manda, etc.
      dir/folders     ---> lista todas las carpetas de un grupo.
      dir/group       ---> lista todos los grupos.

      select <folder> ---> seleciona el folder: <folder>
      select <group>  ---> lo propio con grupos.

      set archive     ---> permite listar todas las news, incluso
     los pasados de fecha.
     Las noticias tienen algo asi como una
     "fecha de caducidad" a partir de la
     cual, no se salen cuando pides el
     listado de news...
     a caso que ejecutes "set archive" antes
     de pedir el listado, en cuyo caso,
     te lista todo.

      read <numero>   ---> permite leer la new que tenga como
    numero de orden: <numero>
      <numero>        ---> igual que el anterior

      extract <file>  ---> escribe todo al fichero <file>
 extract/noheader <f> ---> escribe solo el contenido de la noticia
    al fichero <f>.
      add             ---> anyade una noticia, si el grupo es
    publico.

      up,down,newx,back,top,
      current,bottom  ---> permite movimiento dentro del sistema.

      q               ---> sale de las news


NEWSRDR

 
 
  Es un lector de NEWS interactivo pero no visual, cualquier
informacion deseada hay que solicitarla, como cualquier otro
sistema para leer NEWS es necesario suscribirse a los grupos en
los que se tiene interes.

  Existe comandos para visualizar los grupos existentes, como los
mensajes dentro de cada uno de ellos.

  La informacion del entorno de trabajo y el estado del sistema de
NEWS (grupos suscritos, mensajes leidos, etc) se almacenan en un
fichero en el directorio por defecto de cada usuario. Para evitar
perder esta informacion no se debe modificar o borrar este fichero
(NEWSRDR_PROFILE.NRPF).

  Esta aplicacion se arranca con el comando NEWSRDR, al arrancar se
conecta al servidor de NEWS y recupera la informacion de todos los
grupos de NEWS situandonos en el ultimo que utilizamos en la ultima
sesion. Cuando estas operaciones (que pueden tardar dependiendo de
la cantidad de grupos y noticias existentes) hayan finalizado nos

    B.-Optener Ayuda
    
  La tradicion (por suerte a extingir), es consultar el manual a cada
  instante. Gracias a unos 'maravillosos numerajos', se va a la lista
  de codigos de error y se busca el error que nos salio.

  Ahora se usan mensajes de error mas amigable y la ayuda en contexto
  es profusa. (sale un tipo de ayuda segun en que parte del programa
  estemos)

  Tenemos 3 formas de consegir la ayuda referente a cualquier
  comando/programa :


    # <comando> /?

      * escribir '/?' despues del nombre del programa, y dar enter.
      * Presenta las opciones mas importantes.

    # help <comando/programa a consultar>

      * usar el programa 'help' seguido del item a consultar.
      * Particularidades :
       --- MSDOS ----------------------------------------------------
      | - el comando help usa el entorno del editor y                |
      |   el programa qbasic.                                        |
      | - existe a partir de la version 6.0                          |
       --------------------------------------------------------------

    # fasthelp <comando/programa a consultar>

      * Para una ayuda muy resumida, (de solo una linea!)
        usaremos el comando 'fasthelp'.
      * Particularidades :
       --- MSDOS ----------------------------------------------------
      | - 'fasthelp' existe apartir de la version 6.0                |
      | - en la version 5.0, 'help' realiza esta funcion.            |
       --------------------------------------------------------------





  2. Ejecucion de comandos

    A.-Gestion de Procesos
    

    # <programa>

      * Para ejecutar un programa, solo hay que poner el nombre
        y dar enter. El D.O.S. lo busca en el directorio actual
        o en los directorios de la variable de entorno PATH.
        Si no esta en ambos, habra que poner el nombre completo
        del programa : path\nombre_prog

    # [Unix95] ps

      * Muestra la lista de procesos (tareas) con su corespondiente
        numero asociado ( PID )

    # [Unix95] kill <numero>

      * Mata un proceso de la lista de procesos identificado
        por su numero asociado ( PID )
      * Para parar todos los proceso de golpe,
        Control-Alt-Sup hace maravillas!
        ( es conocido como el saludo de los tres dedos )

    # [Win95] start <prog_win95>

      * Lanza la ejecucion de 'prog_win95' en background.



    

    B.-Interpretes de comandos : 'shells'
    

  El interprete de comandos es el programa que se encarga de
  leer los comandos que pedimos a la maquina, y hacer lo nece-
  sario para que se ejecuten.
     
    
    B.1.-'Shells' mas sencillos
    
    # command.com

      * Interprete de comandos por defecto.
      * Hay que ir tecleando uno a uno los comandos/programas
        que queramos que ejecute.

    # dosshell.exe

      * Programa con un entorno que facilita el manejo de todos los
        comandos del D.O.S., tanto para ver el sistema de fichero,
        como para modificarlo.


    
    B.2.-Como personalizar el 'shell'
    
    Tenemos dos formas de personalizar el shell :
        
        
    B.2.1-VARIABLES DE ENTORNO
    
        Una variable de entorno es una cadena de caracteres que
        hace referencia a otra cadena. Ejemplo, "saludo=hola",
        saludo es la variable de entorno que usaremos para
        referirnos a hola.
        Veamos el uso con ejemplos de referencia :

        # set <<d>>=<<c:\dos>>

	  * Como se definen las variables de entorno ...

        # set

	  * ... como se ver las variables de entorno y su equivalencia ...

        # dir %<<d>>%

	  * ... y como se referencian ...

        # set <<d>>=

	  * ... y como se borran!.


        
    B.2.2-FICHEROS DE CONFIGURACION
    
        Seguramente, las modificaciones echas en el apartado
        anterior querremos que sean permanentes, no tener que
        teclearlas cada vez que arranquemos la cuenta. Como?,
        escribiendo los comandos en un fichero llamado :

        # autoexec.bat

    A.-Navegando en los Directorios
    

    # dir <directorio>
        .  dir /v --> informacion completa
        .  dir /s --> listado recursivo

      * Lista los ficheros del directorio que se le pase como argumento
        ( si no se usa argumentos, lista el directorio en que nos
          encontremos en ese preciso momento )
      * Particularidades :
       --- MSDOS 5.0 -> ------------------------------------------
      | - 'dir /s'                                                   |
      |   lista el contenido del directorio actual y sus             |
      |   subdirectorios                                             |
      |   ( atentos a la tecla de pausa! o a usar                    |
      |     la opcion '/p' tambien )                                 |
       --------------------------------------------------------------

    # cd <c>
        . cd \   ---> ir a la raiz
        . cd ..  ---> ir al anterior

      * Cambia de directorio.
        Realmente, cambia el contenido de una variable del sistema
        operativo en la cual guarda el directorio de trabajo, que es
        aquel que el sistema usara cuando lo omitamos.
      * Particularidades :
       --- Novell ---------------------------------------------------
      |  - 'cd ...' --- 'cd ..\..'                                   |
      |  - 'cd <servidor>\<volumen>:<directorio1\directorio2...>' |
      |     Permite pasar a un directorio de un servidor y volumen en concreto. |
       --- MSDOS 7.0 -> ------------------------------------------
      | - Podemos usar :                                             |
      |     . cd "Este es un directorio especial"                    |
      |     . cd "Este es un directorio especial                     |
      |   para acceder a un directorio por su nombre extendido,      |
      |   o bien usar la nomenclatura tradicional                    |
      |     . cd  Este~1                                             |
      | - 'cd ....' --- 'cd ..\..\..'                                |
      |                                                              |
       --------------------------------------------------------------

    # dir

      * Ademas de listar los ficheros, al principio, indica cual
        es el actual directorio de trabajo.

    B.-Analizando Ficheros
    

    # more < <fichero>
        . ctrl-c            ---> salir
        . cualquier tecla   ---> siguiente pagina

      * Visualiza el contenido de un fichero, parando cuando se
        "llene" una pantalla y esperando a que el usuario presione
        teclas para continuar, o no.
      * Particularidades :      
       --- MSDOS ----------------------------------------------------
      | - tambien vale :                                             |
      |   . 'type msdos | more'                                      |
      |   . 'more <fichero>' (7.0 ->)                       |
       --- DRDOS ----------------------------------------------------
      | - tambien vale :                                             |
      |   . 'type <f> /p'                                      |
       --------------------------------------------------------------

    # type <fichero>

      * Visualizacion fichero sin pausa.

    # [Unix95] tail -<numero> <f>

      * Visualiza las ultimas <numero> lineas del fichero <f>.

    # [Unix95] head -<numero> <f>

      * Visualiza las primeras <numero> lineas del fichero <f>.

    # {fc/comp} <fichero1> <fichero2>

      * Compara el contenido de dos ficheros e indica las
        diferencias que encuentra.
        Se recomienda usar :

           FC   para ficheros ASCII y
           COMP para binarios de identico tamañ.

    # [Unix95] wc -l <fichero>

      * Cuenta el numero de caracteres, palabras y lineas
        del fichero <fichero>.

    # sort <fichero>

      * Ordena items de un fichero.



    

    C.-Busqueda de/en Ficheros
    

    # find <cadena> <fichero>
      [bc2.0->] grep <expresion regular> <fichero>

      * Busca una cadena de caracteres en un fichero.

    # dir <nombre de fichero> /s

      * Usado para buscar donde se encuentra un fichero
      * Particularidades :
        - 'dir <fichero> /s' funciona en :
          . ms-dos->5.0
          . dr-dos->5.0
          . pc-dos->5.0



    

    D.-Facilidades en accesos a Ficheros
    

  1.- CAMINOS DE BUSQUEDA (PATH)
   - Se suele usar una variable de entorno especial que contiene
     una lista de directorios donde buscar donde esta un fichero
     ejecutable cuando solo le damos al interprete de comandos
     el nombre del fichero binario, y no esta en el directorio
     actual (primero en la busqueda).

   - En general es algo asi:
        PATH=c:\dos;c:\

   - Por defecto, esta limitado a 128 caracteres, pero se puede cambiar;
     Para ampliar a 512 caracteres, basta con editar el archivo
     'config.sys' y añadir :
        shell=c:\command.com /e:512



    

    E.-Modificando el Sist. de Fich.
    

  # md <dir name>
    mkdir <dir name>

    * Crea un directorio nuevo, hijo del actual, llamado <dir name>.

  # rd <dir name>
    rmdir <dir name>

    * Borra el directorio <dir name>.
    * '<dir name>' puede ser el path concatenado con
      el nombre del directorio a borrar :

            > rd a:\doks\pp\direct
            ( borra el directorio "direct", hijo de pp )

  # del <nombre fichero>
    erase <nombre fichero>

    * Borra el fichero <nombre fichero>.
    * Igual que al borrar directorios, <nombre fichero> puede
      ser el path y nombre del fichero :

             > del a:\directo\fiche.dos
             borra "fiche.dos", hijo del directorio "directo"

    * Si no te deja borrar el s.o. , es que segun las proteccion,
      no estas autorizado. Cambialas y luego intentalo de nuevo
      (ver seccion protecciones)
    * Es posible "desborrarlo" y recuperarlo con el comando 'undelete *.*',
      si no se ha usado su espacio en disco para otro fichero.
    * Particularidades :
       --- Novell ---------------------------------------------------
      | - 'purge' sirve para que los ficheros borrados NO se puedan  |
      |   recuperar de ninguna forma (si no eres supervisor, claro!) |
       --------------------------------------------------------------

  # copy <nombre fichero origengt; <nombre fichero destinogt;

    * Copia un fichero :
      crea otro fichero que tendra el mismo contenido que el original.
      Si se omite el nombre de fichero destino, y solo se indica el
      directorio donde colocarlo, se usara el mismo nombre que el
      que tiene el fichero origen.

  # ren     <nombre> <nuevo nombre>
    rename  <nombre> <nuevo nombre>
    move    <nombre> <nuevo nombre>

    * Cambia de nombre un fichero.
      'move' puede cambiar tambien el nombre de un directorio.
    * Particularidades :
       --- MSDOS 7.0 -> ---------------------------------------------
      | - ren/rename tambien cambia el nombre de directorios         |
       --------------------------------------------------------------

  # move <path\fichero> <nuevo_path\fichero>

    * Cambia la ubicacion de un fichero.
      (lo traslada a otro directorio borrandolo de su posicion original)
      Puede cambiar, ademas, el nombre del fichero.
      Ciertas versiones, pueden cambiar la ubicacion de un directorio.
    * En los sistemas que no lo implementa se puede simular con la
      secuencia "rename+delete" o "copy+delete".
    * Particularidades :
      - aparece a partir de las siguientes versiones :
        . MSDOS 6.0  ->
        . DRDOS 5.0  ->
        . PCDOS 6.1? ->

  # [Unix95] touch -t MMDDhhmm <fichero>

    * Cambia SOLO la fecha de un fichero;
      La fecha de ultima modificacion del fichero sera 'MMDDhhmm'.
      'MMDDhhmm' es el mes-dia-hora-minuto en numeros y juntos.

  # copy con <f>

    * Edita linea a linea (sin retroceso) un fichero.
      Para terminar, teclear el final de fichero con Ctrl-Z 

  # { edlin | edit | ... <fichero> }

    * Edita un fichero. ( permite cambiar el contenido de un
      fichero, normalmente de texto, claro. )
    * Hay muchos editores, que no pertenecen al sistema operativo
      base, pero son mas potentes y versatiles.
    * Particularidades :     
      - el nombre del editor depende del D.O.S. :
        . 'edit' ----- MSDOS 5.0->
        . 'editor' --- DRDOS (castellano) 5.0->

  # [Unix95] cut -b <bytes> <fichero>

    * Crea los ficheros que necesite, resultado de partir el
      fichero de entrada <fichero> en ficheros de <bytes> bytes
      de tamaño.

  # copy <a> + <b>

    * Concatena el fichero <b> al final del <a>.

    F.2.-Uso atributos
    

  # attrib <fichero>
    [MSDOS 7.0] dir /v

    * Permite ver los "flags" actuales de cada ficheros.
      Si se omite <fichero>, presenta el listado de todos.
    * En 'dir /v', nos fijaremos en la primera columna de la derecha.

  # attrib {+r|-r}{+s|-s}{+h|-h}{+a|-a} <fichero>

    * Cambia los "flags".
      '+' significa activar y '-' desactivar.
      Si se omite un flag, su valor no se altera.





  4. Manejo de Dispositivos
    5. 



    

    A.-Discos
    

    
    A.1.-Acceso a disco
    
    Se consideran 'unidad de discos' a los dispositivos fisicos
    de acceso directo ( floppy, disco duro, cd-rom, ... ) y a las
    particiones logicas que sobre ellos se realizen.
    Cada uno se distingue con una letra del abecedario, seguido de ':'
    Las asignaciones por defecto mas frecuentes :
      - 'a:' para el floppy de la 1a disquetera.
      - 'c:' primer disco duro.
      - 'f:' primera unidad de red ...

    # <letra>:

      * Permiten cambiar al disco identificado por la letra del abecedario
        'letra', y acceder a su informacion.

    # join

      * Une una unidad de disco a un directorio en otra unidad.
        De esa forma, cambiando a ese directorio podemos ver el
        contenido de la unidad. :

           join a: c:\floppya
           ( cada vez que pongas c: y cd floppya es como si
             hubieras puesto a: )

      * '<a:>'
         equivale a :
         'join <a:> <c:\a>' + 'cd \a'

    # subst <unidad_virtual:> <unidad:\path\directorio>

      * Asocia una ruta ( un path, un directorio con su camino
        de aceso completo ) a una unidad 'virtual' :

           subst w: c:\dos\tmp
           ( cada vez que pongas w:, es como cd c:\dos\tmp )

    # subst

      * Lista todas las asociaciones echas con 'subst'

    # subst <unidad_virtual>: /d

      * Borra una asocia echa anteriormente.


    
    A.2.-Manejo de discos
    

    # diskcopy <unidad_origen>: <unidad_destino>:

      * Permite duplicar un Floppy.
        La unidad destino NO necesita estar formateada.


    # format <unidad>:

      * Graba informacion de control necesaria para que D.O.S. pueda
        manejar el disco. Indispensable antes de usar por
        primera vez.



    

    B.-Impresion
    

  # print <fichero(s)> /d:<puerto>

    * Envian un o varios ficheros a la impresora que este conecta
      al puerto <puerto> (LPT1,...,COM!,...)
      ( si se esta imprimiendo alguno, pasa a una cola de espera )
    * Particularidades :
       --- Novell ---------------------------------------------------
      | - poner 'nprint <f>' para imprimir.                    |
       --------------------------------------------------------------

  # print

    * Permite visualizar la cola de impresion
      ( ficheros a espera de ser imprimidos por la impresora )
    * Particularidades :
       --- Novell ---------------------------------------------------
      | - poner 'npq' para ver la cola asociada a la impresora.      |
       --------------------------------------------------------------

  # print <fichero(s)> /C

      * Borra uno o mas trabajos de la cola de impresion
